# Лабораторная работа 3
## Запуск бота для реального использования

## 1. Выбранный способ деплоя
Выбран облачный деплой на Railway, как рекомендуемый и наиболее простой вариант для постоянной работы бота.

## 2. Почему выбран Railway
- Быстрый старт через GitHub-репозиторий.
- Автоматический билд Python-проекта.
- Удобное управление переменными окружения (`BOT_TOKEN`).
- Подходит для круглосуточного запуска.

## 3. Подготовка проекта к деплою
Перед деплоем добавлены/проверены:
- `requirements.txt` для установки зависимостей.
- `Procfile` с командой запуска `worker: python bot.py`.
- `.gitignore` с исключением `.env`, БД и служебных файлов.
- `.env.example` для примера конфигурации.
- Обработка ошибки отсутствия `BOT_TOKEN`.

## 4. Пошаговый деплой на Railway
1. Репозиторий загружен на GitHub.
2. В Railway создан новый проект из GitHub репозитория.
3. В Railway Variables добавлена переменная `BOT_TOKEN`.
4. Railway автоматически выполнил билд и запуск worker-процесса.
5. По логам подтвержден запуск бота без критических ошибок.

## 5. С какими проблемами столкнулся
- При первом запуске на Railway возникла ошибка:
  `RuntimeError: BOT_TOKEN не задан`.
- Причина: переменная окружения не была задана в Railway.
- Решение: добавить `BOT_TOKEN` в Railway Variables и перезапустить деплой.

## 6. Сбор обратной связи от пользователей
Проведено пилотное тестирование на 3 пользователях.

Пользователь 1 (одногруппник):

- Понравилось: быстрые ответы, удобная конвертация
- Замечание: хотелось поддержку английского языка

Пользователь 2 (друг):

- Понравилось: вывод изменения цены за 24 часа
- Замечание: не сразу понял, какие команды есть

Пользователь 3 (знакомый):
- Использовал: `/start`, `/help`, `/rate`
- Понравилось: простой UX и понятные команды
- Замечание: хотелось более дружелюбные сообщения об ошибках

## 7. Улучшения на основе фидбека
На основе обратной связи выполнены улучшения:

1. Проблема: отсутствовала мультиязычность.
- Что изменено: добавлена локализация RU/EN и команда `/lang <ru|en>`.
- Какой эффект: пользователи могут выбрать удобный язык интерфейса.

2. Проблема: пользователи не сразу находили все команды.
- Что изменено: обновлен `/help`, добавлена команда `/lang`, улучшены подсказки по использованию.
- Какой эффект: сократилось число ошибок при вводе команд.

3. Проблема: сообщения об ошибках были недостаточно дружелюбными.
- Что изменено: унифицированы и улучшены тексты ошибок в командах.
- Какой эффект: пользователю понятнее, что делать дальше.
