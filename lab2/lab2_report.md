# Лабораторная работа 2
## Подключение бота к данным

## 1. Описание интеграции
В бота добавлена интеграция сразу с несколькими источниками данных:
- Публичный API валют (fiat).
- Публичный API криптовалют (CoinGecko).
- Локальная SQLite база для хранения списка наблюдения пользователя.

## 2. Какой источник данных выбран и почему
- `open.er-api.com` для фиатных валют, потому что бесплатный и не требует API ключа.
- `CoinGecko API` для криптовалют, потому что популярный и стабильный.
- `SQLite` для локального хранения пользовательских данных без отдельного сервера.

## 3. Исходный промпт
```text
Улучши моего Telegram-бота, добавив работу с внешними данными.
Нужно:
- получать курсы фиатных валют через API
- получать цены криптовалют через API
- хранить список наблюдения пользователя в SQLite
- сохранить существующие команды и добавить обработку ошибок
```

## 4. Финальный промпт
```text
Обнови Telegram-бота на Python:
- вынеси API-логику в отдельный модуль
- добавь функции get_exchange_rate, get_crypto_price и convert_currency
- сделай SQLite-модуль с init_db, add_to_watchlist, remove_from_watchlist, get_watchlist
- добавь кэширование запросов, чтобы уменьшить нагрузку на API
- сделай понятные ответы при ошибках и недоступности внешних сервисов
```

## 5. Реализация
- `api_client.py`:
  - работа с внешними API,
  - кэширование в памяти (TTL),
  - функции курсов, цен и конвертации.
- `database.py`:
  - создание таблицы `watchlist`,
  - операции добавления/удаления/чтения.
- `bot.py`:
  - интеграция API и БД в команды бота.

## 6. Ключевые фрагменты функционала
- `/rate USD` -> получение курса к RUB.
- `/crypto BTC` -> цена и изменение за 24 часа.
- `/convert 100 USD EUR` -> конвертация.
- `/watch BTC` и `/watchlist` -> персональный список наблюдения.

## 7. Тестирование
Проверены сценарии:
- валидные команды с корректными кодами;
- невалидные валюты/тикеры;
- пустые аргументы;
- повторное добавление тикера в список;
- недоступность внешнего API (обработка исключений).

## 9. Трудности и решения
- Проблема: разные форматы данных у API.  
  Решение: унифицировал формат ответа через отдельные функции-клиенты.
- Проблема: лишние запросы к API при частом использовании.  
  Решение: добавил in-memory кэш.
