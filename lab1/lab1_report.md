# Лабораторная работа 1
## Создание первого Telegram-бота без программирования

## 1. Описание задачи
Для первой лабораторной был выбран бот-ассистент для отслеживания финансовых показателей: курсы фиатных валют и криптовалют.  
Бот должен помогать пользователю быстро получать курс, конвертировать суммы и вести список наблюдения тикеров.

## 2. Какую проблему решает бот
- Экономит время на проверку курсов в разных приложениях и сайтах.
- Позволяет получать данные прямо в Telegram по простым командам.
- Дает базовый персональный инструмент для мониторинга рынка.

## 3. Исходный промпт для LLM
```text
Создай Telegram-бота на Python с использованием библиотеки python-telegram-bot.

Функционал бота:
- Команды /start и /help
- Команда /rate <код> для курса валюты
- Команда /crypto <тикер> для цены криптовалюты
- Команда /convert <сумма> <из> <в> для конвертации
- Команды /watch, /unwatch, /watchlist для списка наблюдения

Требования:
- Бот должен быть простым и понятным
- Добавить обработку ошибок
- Хранить данные в SQLite
- Конфигурацию хранить через переменные окружения

Создай:
1. bot.py
2. requirements.txt
3. README.md
4. .env.example
```

## 4. Финальный промпт
```text
Улучши Telegram-бота для курсов валют и криптовалют на Python.
Сохрани простой стиль кода и добавь:
- асинхронные HTTP-запросы
- стабильную обработку ошибок API
- команды: /rate, /crypto, /convert, /watch, /unwatch, /watchlist
- хранение watchlist в SQLite
- логирование и понятные сообщения пользователю
- файл .env.example и README с инструкцией запуска
```

## 5. Используемый стек
- Python 3.11
- `python-telegram-bot`
- `httpx`
- `sqlite3`
- `python-dotenv`

## 6. Что реализовано в результате
- Рабочий Telegram-бот с командами `/start`, `/help`, `/rate`, `/crypto`, `/convert`, `/watch`, `/unwatch`, `/watchlist`.
- Работа с внешними источниками данных для фиата и криптовалют.
- Персистентное хранение watchlist пользователя в SQLite.
- Базовая обработка ошибок и логирование.
